# Contributing to The Index

Thank you for your interest in contributing to The Index! This document outlines the rules and guidelines for adding entries.

## Rules for Adding Entries

### 1. Entry Structure
All entries must follow the standardized format defined in [INDEX.md](INDEX.md):
- **Term**: The name or concept being defined
- **Definition**: A clear, concise explanation
- **Verifiable Claim**: A factual statement that can be independently verified
- **Example**: A practical illustration
- **Cross-links**: Related terms within the index

### 2. Quality Standards
- **Accuracy**: All information must be factually correct and verifiable
- **Clarity**: Definitions should be clear and accessible to a general audience
- **Neutrality**: Maintain an objective, unbiased tone
- **Citations**: Include sources for verifiable claims when applicable

### 3. Formatting Requirements
- Use Markdown formatting consistently
- Follow the existing entry structure in INDEX.md
- Ensure proper spelling and grammar
- Use appropriate heading levels

### 4. Verification
- Every verifiable claim must be supported by evidence
- Include references to authoritative sources when possible
- Claims should be independently verifiable by others

### 5. Cross-linking
- Link to related terms within the index using relative links
- Ensure cross-referenced terms exist or are being added simultaneously
- Maintain bidirectional links when appropriate

### 6. Submission Process
1. Fork the repository
2. Create a new branch for your entry
3. Add your entry following the format in INDEX.md
4. Submit a pull request with a clear description
5. Be responsive to review feedback

### 7. Review Criteria
Pull requests will be reviewed for:
- Adherence to the entry format
- Accuracy and verifiability of information
- Quality of writing and clarity
- Appropriate cross-linking
- Overall value to the index

## Getting Help

If you have questions about contributing, please open an issue for discussion before submitting your entry.
