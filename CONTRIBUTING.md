# Contribution Guidelines

All contributions must follow the entry format defined in `INDEX.md`.

## Rules
1. Each entry must contain a **verifiable claim**.  
2. Definitions must be **precise**, **global**, and **non‑cultural**.  
3. Examples must be **concrete**, not abstract.  
4. Cross‑links must reference existing or newly added entries.  
5. Commit messages must follow this pattern:
   - `docs:` for documentation changes  
   - `chore:` for structural or organizational changes  
   - `feat:` only if adding new indexing capabilities  

## Process
- Add or modify entries in `INDEX.md`.  
- Submit a pull request with a clear description.  
- Ensure all entries maintain internal consistency.
